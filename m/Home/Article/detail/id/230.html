数据结构的定义

关于数据结构的定义并没有统一，不过按照中文维基百科上的解释比较好理解一些。

数据结构（data structure）是计算机中存储、组织数据的方式。通常情况下，精心选择的数据结构可以带来最优效率的算法


通过摆放图书的例子理解数据结构

如何在书架上摆放图书？

如何在书架上查找图书？

如何在书架上新增图书？


可以随便放，随地新增，但是图书一旦很多查找很要命。

可以按字母顺序放，查找通过二分法查找。这样大数据下查找也会很要命。

再优化一下就是按字母分类放，之后在这个字母的分类里面进行添加，查找，相对于在全部书里面查找快速很多。


把这个问题联想到实际生活中书店的书是怎么摆放的？


得出结论：解决问题方法的效率跟数据的组织方式有关。


写程序实现一个函数PrintN，使得 传入一个正整数为N的参数后，能顺序打印从1到N的全部正整数

//循环实现
voidPrintN( intN ) 
{ 
    int i; 
    for( i=1; i<=N; i++ )
    { 
        printf(“%d\n”, i ); 
    } 
    return; 
}
//递归实现
voidPrintN( intN ) 
{ 
    if( N )
    { 
        PrintN( N –1 ); 
        printf(“%d\n”, N ); 
    } 
    return; 
}

当N较大是会发现递归实现的算法会爆炸，因为占用了过多的内存资源，而循环实现只是对同一个内存地址N次赋予不同的值，不会因为N的次数增大而增大保存N的内存资源。


得出结论：解决问题方法的效率，跟空间的利用效率有关。


再比如，写程序计算一个多项式

//方法1
doublef1( int n, double a[], double x )
{
	inti; double p = a[0]; for ( i = 1; i <= n; i++ )
		p += (a[i] * pow( x, i ) );
	return p;
}
//方法2
doublef2( int n, double a[], double x )
{
	int i; double p = a[n]; for ( i = n; i > 0; i-- )
		p = a[i - 1] + x * p;
	return p;
}

显然方法二更加“巧妙”。如何直观的表示这种巧妙呢？

简单来讲，在不考虑占用资源的情况下，根据算法执行的快慢，也就是时间长短来衡量算法的优劣。

假如一个算法运行时间极短又该怎么判断时间呢？可以运行N次，取个平均嘛！

得出结论：解决问题方法的效率，跟算法的巧妙程度有关。


总结

数据对象在计算机中的组织方式：逻辑结构，物理存储结构。

数据对象必定与一系列加在其上的操作相关联。

完成这些操作所用的方法就是算法。
